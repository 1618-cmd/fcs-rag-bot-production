Data Export Endpoints

GET Requires Model ID
/models/{modelID}/intersections
Get intersection data from a Vena model. This is the primary endpoint for exporting actual data values.
Parameters:
  • modelID - The model ID (required, get from Vena UI)
  • filters - JSON array of filters (optional)
    Example: [{"field":"Year","eq":"2023"}]
  • fields - Comma-separated list of fields to return (optional)
Example URL: https://eu1.vena.io/api/public/v1/models/{modelID}/intersections?filters=[{"field":"Year","eq":"2023"}]

Model Structure Endpoints

GET
/models/{modelID}
Get model structure and metadata. Returns information about a specific data model including dimensions and hierarchies.
Parameters:
  • modelID - The model ID (required)
GET
/models
List all available models. Returns a list of data models in the tenant.

Dimension Endpoints

GET
/models/{modelID}/dimensions
Get all dimensions for a model. Returns list of dimensions (Account, Entity, Period, Measure, etc.)
Parameters:
  • modelID - The model ID (required)
GET
/models/{modelID}/dimensions/{dimensionName}/members
Get members of a specific dimension. Returns all members (e.g., all Account codes, all Entities)
Parameters:
  • modelID - The model ID (required)
  • dimensionName - Name of dimension (e.g., "Account", "Entity", "Period")

ETL & Integration Endpoints

GET
/templates
Get list of ETL templates. Returns all ETL templates configured in the tenant.
GET
/jobs
Get list of ETL jobs. Returns ETL job execution history.
Parameters:
  • templateId - Optional template ID to filter jobs

What We Can Access

  • Real-time data values - Actual numbers from the Vena model (revenue, expenses, etc.)
  • Model structure - Dimensions, hierarchies, and metadata
  • Dimension members - All accounts, entities, periods, scenarios, etc.
  • ETL information - Templates and job history
  • Filtered queries - Get data for specific years, entities, accounts, etc.

How to Get a Model ID

  1. Log into Vena at https://eu1.vena.io
  2. Navigate to Modeler tab
  3. Select Data Model from the sidebar
  4. Click on a data model name
  5. The Model ID appears in:
    • The URL (look for a long ID in the URL path)
    • Model details page (usually shown at the top)
